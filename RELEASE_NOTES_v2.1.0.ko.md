## 한 눈에 보기 (TL;DR)

다 못 읽으셔도 이 세 가지만 알아두세요:

1. **Mode E가 새로 생겼습니다.** 어닝 발표일 전후 7일 윈도우 전용 모드입니다. 발표 *전*에는 무엇을 지켜봐야 할지, 발표 *후*에는 무엇이 실제로 바뀌었는지를 — 애널리스트가 아니라 **일반 투자자가 이해할 수 있는 언어로** 알려줍니다.
2. **모든 보고서 최상단에 "지난번 대비 변화" 배너가 자동으로 뜹니다.** AAPL, NVDA, 또는 삼성전자를 다시 분석하면 R/R 점수, 목표가, Verdict, 리스크 리스트의 차이를 즉시 볼 수 있습니다.
3. **Mode C 보고서가 5가지 측면에서 업그레이드됐습니다.** Peer 표가 placeholder가 아닌 실제 숫자로 채워집니다. 12개월 카탈리스트 타임라인이 추가됐습니다. DCF / Peer 멀티플 / 애널리스트 컨센서스 / 우리 base 시나리오 4개 anchor를 한 차트에서 조정하는 "Valuation Bridge"가 생겼습니다. 그리고 Mode B 비교에 매크로 컨텍스트가 추가되어, 같은 섹터 종목들이 *왜* 금리 변화에 다르게 반응하는지 알 수 있습니다.

이 중 하나라도 유용해 보이면 아래 본문을 읽어주세요.

## 새로 추가된 기능

### 1. Mode E — 실적 프리뷰 & 리뷰 (메이저 신규)

어닝 윈도우 전용 분석 모드입니다. 어닝 발표일 기준으로 어느 쪽에 있는지 **자동 감지**합니다:

| 윈도우 | 모드 | 제공 내용 |
|---|---|---|
| **D-7 ~ D-1** (발표 전 일주일) | **Preview** | 컨센서스 EPS·매출, 옵션시장 implied 1일 변동폭, 마지막 8분기 beat/miss 히스토리, 이번 발표에서 지켜볼 4-5개 핵심 watchpoint 질문, pre-mortem 시나리오, 사전 포지션 권고 |
| **D ~ D+3** (발표 당일과 이후 3일) | **Review** | 실제 vs 컨센서스 표, 가이던스 변화 (FY EPS pre/post), Preview에서 던졌던 watchpoint 질문에 대한 답, thesis 영향, 가벼운 verdict 업데이트 + Mode C 재실행 권고, 사후 액션 (Hold / Trim / Hedge / Add) — 진입·청산 levels 포함 |

**왜 중요한가**

어닝 윈도우는 개인 투자자가 가장 많이 손실을 보는 구간입니다 — 발표 직전에 사고, 직후에 패닉으로 던지고, 정작 가이던스에 담긴 진짜 신호는 놓칩니다. Mode E는 정확히 그 순간을 위해 만들어졌습니다.

**실제 예시** — AMD Q1 2026 발표 D+2 시점: 데이터센터 부문 어닝 비트로 +18.61% 폭등한 다음날이었습니다. 일반 Mode C 보고서라면 "주가 상승, 멀티플 확장됨" 정도로 끝날 상황이었습니다. Mode E Review는 어떤 사업부가 폭등을 견인했는지 (데이터센터 매출 YoY +57%), 연간 가이던스가 어떻게 바뀌었는지, 우리가 발표 전에 던졌던 watchpoint 질문 중 어떤 것이 실제로 결정적이었는지, 그리고 +18% 폭등을 따라가야 하는지 페이드해야 하는지를 분해해서 보여줬습니다.

**사용법**

어닝 발표일 7일 이내 시점에 `AMD 어닝 분석해줘` 또는 `AMD 실적 분석`이라고 말하면 됩니다. 에이전트가 윈도우를 자동 감지해서 Preview 또는 Review를 선택합니다.

**접근성 레이어 (2026-05-07 추가)**

Mode E를 실제 어닝콜에서 처음 테스트했을 때 사용자 피드백은 *"compact하고 전문가가 아니면 보기 어렵다"* 였습니다. 그래서 모든 Mode E 보고서에 4가지를 추가했습니다:

- **📌 최상단 TL;DR** — 3개 핵심 bullet, 30초 만에 결론 파악
- **📊 사업부별 매출 분해 표** — Data Center / Client / Gaming 등 segment별로 매출, YoY %, 매출 비중, 영업이익률
- **💡 "일반 투자자 입장에서" 콜아웃** — 각 섹션마다 *왜 이 숫자가 중요한지* 평이한 한국어 설명
- **📚 용어 풀이** — Surprise %, Forward P/E, TAM, Multiple Re-rating 등 jargon 10개를 본문 하단에서 plain Korean으로 정의 — 검색하지 않고도 끝까지 읽을 수 있음

보고서 크기가 22KB → 33KB로 늘어났지만, 빈 셀이 모두 채워졌고 비전문가도 끝까지 따라갈 수 있게 됐습니다.

### 2. Auto Delta 배너 — 모든 보고서가 "지난번"을 기억합니다

**이전** — 매 분석이 독립된 스냅샷이었습니다. 지난주에 NVDA를 분석하고 오늘 다시 분석하면, 두 보고서를 손으로 비교해야 무엇이 달라졌는지 알 수 있었습니다.

**이후** — 모든 보고서가 최상단에 delta 배너로 4가지를 자동 표시합니다:

- **R/R 점수 변화** — 예: "1.42 → 1.69 (+0.27)"
- **목표가 변화**
- **Verdict 변화** (예: Hold → Buy)
- **리스크 리스트 diff** — 새로 추가된 리스크, 해소된 리스크

**사용법**

별도 작업 필요 없습니다. 이전 스냅샷이 존재하는 종목을 재분석하면 자동으로 실행됩니다. 특정 분석에서 끄고 싶다면 `--no-delta` 플래그를 추가하세요.

**왜 중요한가** — 주식 리서치에서 가장 어려운 질문은 "thesis가 진짜로 바뀐 건가, 아니면 내가 노이즈에 반응하는 건가?"입니다. Delta 배너가 그 답을 강제합니다 — R/R이 0.05만 움직였다면 노이즈 트레이딩이고, 0.50 움직이고 신규 리스크가 3개 추가됐다면 thesis가 실제로 변한 것입니다.

### 3. Mode B — Peer 비교에 매크로 컨텍스트 추가

**이전** — 2종목 Mode B 비교 (예: 삼성전자 vs SK하이닉스)는 P/E, 마진, 성장률을 나란히 보여줬습니다. 하지만 두 종목이 매크로 환경 변화에 *왜* 다르게 반응하는지는 보여주지 않았습니다.

**이후** — Mode B에 light 매크로 컨텍스트 블록이 추가됐습니다: 회사 type별 핵심 매크로 시리즈 3-5개, 그리고 종목별 매크로 노출 차이 narrative.

**실제 예시** — 삼성전자 (Beta 1.3) vs SK하이닉스 (Beta 2.0): 새 매크로 블록은 +50bp 금리 인상 시 SK하이닉스가 역사적으로 삼성전자 대비 약 1.5배 움직인다는 사실과 그 이유를 알려줍니다 (HBM 집중도, 고객사 구성, 재무 레버리지). *"둘 다 메모리 종목이니 비슷하겠지"* 와 *"이 둘은 절대 같은 베팅이 아니다"* 의 차이입니다.

**사용법**

기존과 동일합니다: `삼성전자 vs SK하이닉스 비교`. Mode B 출력에 매크로 블록이 자동으로 포함됩니다.

### 4. Mode C — Peer 미니 파이프라인 (placeholder가 아닌 실제 숫자)

**이전** — NVDA에 대한 Mode C 보고서의 "Peer 비교" 표에서 MSFT / META / AMZN / AAPL은 `[Est]` placeholder였습니다. 우리가 subject ticker(NVDA)에 대해서만 실제 데이터를 수집했기 때문입니다.

**이후** — Peer ticker도 yfinance로 abbreviated fetch (peer 당 5-7개 metric, 24시간 캐시)를 거쳐 실제 Portal Grade 숫자로 표에 들어갑니다.

이제 다음과 같이 표시됩니다:

- 각 peer의 실제 P/E, EV/EBITDA, 영업이익률, 매출 성장률, 매출총이익률
- 특정 peer 데이터를 가져오지 못했을 경우 명확한 `⚠️` 표시 (숫자를 지어내지 않습니다 — *빈칸 > 틀린 숫자* 원칙은 그대로)
- 24시간 peer 캐시로 같은 분석을 짧은 간격으로 다시 돌려도 불필요한 재수집 없음

**왜 중요한가** — Peer 비교는 thesis 다음으로 중요한 섹션입니다. 그 숫자들이 추정치라면 비교 자체가 픽션이 됩니다. 이제 모두 실제 숫자입니다.

---

### 5. Mode C — 12개월 카탈리스트 타임라인

**이전** — "예정 카탈리스트" 섹션은 시간순 5개 항목 bullet 리스트였습니다. 훑어보기는 쉬웠지만, 이벤트 cluster를 발견하기는 어려웠습니다.

**이후** — 12개월 Gantt-style 시각화 타임라인 + 5개 색상 카테고리:

- **실적** (예: Q2 발표, Q3 발표)
- **규제** (예: FDA 결정, 반독점 판결, FOMC)
- **제품** (예: 아이폰 발표, GTC 키노트, Investor Day)
- **매크로** (예: CPI 발표, 고용지표)
- **기타** (예: 법적 판결, 배당 발표)

Phase D의 peer 미니 파이프라인이 peer 데이터를 채웠다면, **peer 카탈리스트도 같은 타임라인에 표시됩니다** — 그래서 subject 종목의 어닝이 peer 어닝과 같은 주에 몰려있는지, 또는 어떤 규제 이벤트가 여러 보유종목에 동시에 영향을 미치는지가 한눈에 보입니다.

**왜 중요한가** — 카탈리스트 클러스터링은 실제 리스크입니다. NVDA 어닝 직전 주에 NVDA를 사면서 그 주에 MSFT 어닝이 있고 *동시에* FOMC가 있다면, 그건 3개의 독립적 결정이 아니라 사실 하나의 큰 매크로 베팅입니다. 타임라인이 그 cluster를 가시화합니다.

### 6. Mode C — Valuation Bridge 위젯

**이전** — Mode C 보고서가 동시에 DCF 적정가 -38% (하락) 와 base 시나리오 목표가 +7% (상승) 을 표시할 수 있었습니다. 둘 다 다른 가정 하에서는 기술적으로 맞지만, 보고서가 둘을 reconcile하지 않았습니다. 독자가 알아서 선택해야 했습니다.

**이후** — 새로 추가된 "Valuation Bridge" 섹션이 4개 anchor를 가중평균 적정가 하나로 reconcile하고, 가중치 선택 이유를 paragraph로 설명합니다:

| Anchor | 의미 |
|---|---|
| **DCF 적정주가** | 우리 base 가정 하의 DCF |
| **Peer 멀티플 적정주가** | Peer 중앙값 P/E, EV/EBITDA 적용 시 함의된 가격 |
| **애널리스트 컨센서스 목표가** | Street 중앙값 12개월 목표가 |
| **우리 base 시나리오 목표가** | 자체 시나리오 가중 12개월 목표가 |

위젯은 4개 anchor가 현재가 대비 어디에 있는지 시각적으로 보여주고, 가중평균으로 수렴시킨 뒤 *왜* 그 가중치를 골랐는지를 paragraph로 설명합니다 (예: "DCF는 terminal value 민감도가 높아 가중치 낮춤; peer 멀티플은 peer set이 대칭적이라 가중치 높임").

**왜 중요한가** — Valuation 섹션에서 가장 흔한 독자 불만은 "숫자들이 서로 모순되는데 아무도 인정 안 한다"입니다. Valuation Bridge는 모순을 인정하고, 그 자리에서 해결합니다.
