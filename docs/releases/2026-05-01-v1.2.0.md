# v1.2.0 Release Notes

## English

v1.2.0 sharpens the valuation step and clears two everyday papercuts: macro data that quietly stayed disconnected, and Mode D memos that crashed at the very last step.

### What Improved

- **Reverse DCF — make the market's growth assumption explicit**
  Mode C dashboards and Mode D memos now answer a question the agent was previously hand-waving: "what FCF growth rate is the market actually pricing in at this stock price?" The new Reverse DCF block solves the standard DCF backwards and reports the implied annual FCF growth rate, the gap versus the analyst's Base scenario, and a one-line verdict. This gives a single, falsifiable number to challenge or confirm the Base case — instead of vague "valuation feels rich" framing.

- **Macro data finally auto-loads from `.env`**
  FRED (US macro) and DART (Korean filings) collectors now automatically read their API keys from the project's `.env` file via `python-dotenv`. Previously the keys were sitting in `.env` but Python subprocesses could not see them, so analyses silently fell back to qualitative-only macro context. The fix means Korean and US macro data flows in as Grade A on every run with no extra setup.

- **Mode D DOCX no longer crashes on the footer**
  Generating a Mode D investment memo failed with a `NameError` on the very last step (writing the document footer). The script now reads the data mode field correctly, so memos save reliably without a manual workaround.

- **Bilingual README reflects current capabilities**
  Both English and Korean READMEs explicitly mention the new Reverse DCF block in the Mode C and Mode D feature tables, so readers can see what each output mode actually delivers.

### Why It Matters

The Reverse DCF turns one of the most subjective parts of a valuation — "is this expensive?" — into a concrete number that can be tested against fundamentals. The macro auto-load fix removes a silent failure mode that was capping confidence on every Mode C/D analysis. Together these are small but high-leverage improvements: better reasoning quality, fewer surprises, less manual setup.

## 한국어

v1.2.0은 밸류에이션 단계를 더 날카롭게 만들고, 매크로 데이터 자동 연결과 Mode D 메모 마지막 단계 크래시 두 가지 일상적 불편을 해결한 릴리즈입니다.

### 개선된 점

- **Reverse DCF — 시장이 가정하는 성장률을 숫자로 드러냄**
  Mode C 대시보드와 Mode D 투자 메모가 이전에는 모호하게 넘어가던 질문에 답합니다: "현재 주가에서 시장은 실제로 몇 %의 FCF 성장률을 가정 중인가?" 새로 추가된 Reverse DCF 블록은 표준 DCF를 역방향으로 풀어서 시장이 내재 가정하는 연 FCF 성장률, 분석가의 Base 시나리오와의 갭(gap), 한 줄 verdict까지 보여줍니다. "비싸 보인다"는 모호한 프레이밍 대신, Base 케이스를 검증하거나 반박할 수 있는 검증 가능한 단일 숫자가 생겼습니다.

- **매크로 데이터가 드디어 `.env`에서 자동 로드됨**
  FRED(미국 매크로)와 DART(한국 공시) 수집기가 이제 프로젝트의 `.env` 파일에서 API 키를 `python-dotenv`로 자동으로 읽습니다. 이전에는 키가 `.env`에 있어도 Python 서브프로세스가 인식하지 못해 분석이 조용히 정성적 매크로로만 떨어졌습니다. 이번 수정으로 한국·미국 매크로 데이터가 매 실행마다 별도 설정 없이 Grade A로 흘러 들어옵니다.

- **Mode D DOCX 푸터 크래시 수정**
  Mode D 투자 메모 생성이 마지막 단계(문서 푸터 작성)에서 `NameError`로 크래시했습니다. 스크립트가 데이터 모드 필드를 올바르게 읽도록 수정해, 별도 우회 작업 없이 메모가 안정적으로 저장됩니다.

- **README가 현재 기능을 정확히 반영**
  영어·한국어 README 모두 Mode C와 Mode D 기능 표에 신규 Reverse DCF 블록을 명시했습니다. 각 출력 모드가 실제로 제공하는 내용을 명확히 확인할 수 있습니다.

### 왜 중요한가

Reverse DCF는 밸류에이션에서 가장 주관적이던 부분 — "이 가격이 비싼 건가?" — 을 펀더멘털로 검증 가능한 구체적 숫자로 바꿉니다. 매크로 자동 로드 수정은 모든 Mode C/D 분석에서 신뢰도를 조용히 낮추던 silent failure를 제거합니다. 작지만 레버리지가 큰 개선들입니다: reasoning 품질 향상, 예기치 않은 결함 감소, 수동 설정 부담 감소.
